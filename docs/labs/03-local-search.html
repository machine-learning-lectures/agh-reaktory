Lab 3: Local Search dla tras ekip

Pytanie przewodnie

Jak poprawiać kompletne rozwiązanie bez budowania całego modelu MILP?

Cel laboratorium

Po tym labie student powinien rozumieć:

  1. Różnicę między rozwiązaniem konstrukcyjnym a poprawiającym.
  2. Czym jest sąsiedztwo rozwiązania.
  3. Dlaczego local search może szybko poprawić trasę, ale nie daje gwarancji globalnego optimum.

4. Wizualizacja

def plot_route(ax, route, title):
    xs = [points[node][0] for node in route] + [points[route[0]][0]]
    ys = [points[node][1] for node in route] + [points[route[0]][1]]
    ax.plot(xs, ys, marker="o", linewidth=1.8)
    for node in route:
        x, y = points[node]
        ax.text(x + 0.08, y + 0.08, node, fontsize=8)
    ax.set_title(title)
    ax.set_aspect("equal", adjustable="box")
    ax.grid(True, alpha=0.25)

fig, axes = plt.subplots(1, 2, figsize=(11, 4))
plot_route(axes[0], nn_route, f"Nearest neighbor: {nn_cost:.2f}")
plot_route(axes[1], improved_route, f"2-opt: {improved_cost:.2f}")
plt.show()

Interpretacja

Local search nie szuka w całej przestrzeni. Szuka w sąsiedztwie aktualnego rozwiązania. Ta prostota jest jego siłą i słabością: kod jest szybki, ale zatrzymuje się, gdy żadna pojedyncza operacja 2-opt nie poprawia trasy.
