Lab 1: Greedy jako pierwszy plan postoju

Pytanie przewodnie

Kiedy prosta reguła „bierz najbardziej opłacalną pracę jako pierwszą” daje sensowny plan, a kiedy produkuje elegancką pomyłkę?

Cel laboratorium

Po tym labie student powinien rozumieć trzy rzeczy:

  1. Algorytm zachłanny jest świetnym baseline’em, bo jest szybki i interpretowalny.
  2. Poprawność greedy zależy od struktury problemu, nie od urody implementacji.
  3. W problemach 0-1 zaokrąglanie intuicji z wersji ciągłej bywa niebezpieczne.

1. Syntetyczny backlog prac

Zaczynamy od listy prac remontowych. Każda praca ma czas trwania, koszt wykonania, wartość unikniętego opóźnienia i przybliżoną dawkę wynikającą ze strefy.

3. Czy wykorzystaliśmy zasoby?

import matplotlib.pyplot as plt

usage = pd.DataFrame(
    {
        "resource": ["czas", "budżet", "dawka"],
        "used": [
            greedy["total_duration"],
            greedy["total_budget"],
            greedy["total_dose"],
        ],
        "limit": [
            DEFAULT_LIMITS.max_duration,
            DEFAULT_LIMITS.max_budget,
            DEFAULT_LIMITS.max_dose,
        ],
    }
)
usage["share"] = usage["used"] / usage["limit"]

ax = usage.plot.bar(x="resource", y="share", legend=False, color="#3366aa")
ax.axhline(1.0, color="black", linewidth=1)
ax.set_ylim(0, 1.1)
ax.set_ylabel("wykorzystanie limitu")
ax.set_title("Greedy: wykorzystanie ograniczeń")
plt.show()

4. Kontrprzykład

Greedy wygląda rozsądnie, dopóki nie pokażemy małej instancji, w której wybór lokalnie najlepszy blokuje lepszą parę prac.

from utilis.library import make_greedy_counterexample

counter_tasks, counter_limits = make_greedy_counterexample()
counter_df = pd.DataFrame(counter_tasks)
counter_df["value_per_duration"] = counter_df["value"] / counter_df["duration"]
